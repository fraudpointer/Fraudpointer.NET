 ClientException (string message, Exception innerException)

Detailed Description

Thrown by all IClient methods in case an error takes place and method cannot return its results normally.

You should always put your IClient methods into try/catch blocks and make sure you catch this exception. If you catch this exception, the message borne will be adequate to explain to you what has gone wrong. Especially the message borne by the InnerException of the Exception caught.

Here is how you can wrap your code (example of a Windows Console Application) arround try/catch block and on error get information about what has gone wrong: 

        try
        {
           ... write your FraudPointer code here ....

        } // try
        catch (Fraudpointer.API.ClientException ex)
        {

           // something went really wrong. Let us find out what:
           Console.Error.WriteLine(ex.Message);
           // and the inners of it:
           Console.Error.WriteLine("Inner: {0}", ex.InnerException.Message);

        } // catch
